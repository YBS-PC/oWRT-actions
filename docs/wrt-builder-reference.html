Справочник прошивки

Universal WRT Builder

Конвейер GitHub Actions, который собирает прошивку OpenWrt / ImmortalWrt под несколько устройств в выбранном «варианте» начинки, и скрипт zz1-final-offline-setup.sh, который доводит роутер до рабочего состояния на первом старте. Эта страница объясняет, что задаётся при запуске, что происходит во время сборки, чем отличаются варианты и что получается после прошивки.

02

Конвейер сборки

Что делает Universal_WRT_Builder.yml по шагам. Сначала job setup строит матрицу из выбранного target/variant, затем job build на раннере ubuntu-22.04 прогоняет сборку для каждой пары «устройство × вариант».

  1. Матрица сборки. Из target/variant формируется список пар: combo → три фикс-сборки, all → r6s+rb5009, одиночный → одна пара.
  2. Подготовка раннера. Освобождение места на диске, checkout репозитория со скриптами.
  3. Права на files/. Делаются исполняемыми скрипты в files/etc/uci-defaults и files/usr/bin (попадут в образ).
  4. Скачивание AdGuardHome — если включён флаг и вариант это допускает. Бинарь кладётся в files/.
  5. Скачивание sing-box — если выбрано не no и вариант это допускает.
  6. hp_hiddifyПока не добавлен. Скачивание пакетов hiddify. Только для hp_hiddify: готовые подписанные пакеты движок hiddify-core + luci-app + RU i18n + ключ кладутся в files/root/apps/.
  7. Переменные сборки. Вычисляются URL репозитория, ветка, источник/коммит; тег релиза ищется через GitHub API (с фолбэками); подгружается конфиг устройства config/conf/<target>.
  8. Зависимости (apt). Установка тулчейна сборки. Из-за python2.7 раннер закреплён на ubuntu-22.04.
  9. Клон исходников и checkout на нужный коммит/тег OpenWrt/ImmortalWrt.
  10. Патчи. Применяются кастомные патчи из каталога patches/.
  11. Update Feeds & Config. Главный шаг: P1 → feeds update → P2 → feeds install → копирование WRT.config в .config → команды устройства → P3 → правки под вариант → P4 → удаление по Exclude-словам → make defconfig.
  12. Kernel Hash. Хеш ядра подменяется на официальный, чтобы на роутер можно было ставить готовые kmod из репозиториев.
  13. Компиляция. Многопоточная; при сбое — повтор в один поток с сохранением build.log.
  14. Bootloader-файлы. Глубокий поиск и сохранение idbloader.img/u-boot.itb/.fip для прошивки загрузчика.
  15. Артефакт. Образ загружается (upload-artifact@v7), старые прогоны workflow подчищаются.

Колонка hp_hiddify — опциональный вариант (пока не добавлен): стоковый homeproxy и sing-box убираются, а движок hiddify-core с веб-интерфейсом и RU-локализацией ставятся готовыми подписанными пакетами на первом старте (см. раздел 04). Различие clear и crystal_clear: crystal_clear вдобавок удаляет сам zz1 — это «чистая болванка» без первичной настройки. switch делит bloat-список с crystal_clear, но оставляет zz1, который переводит устройство в L2-мост.

04

Первый старт после прошивки

Скрипт zz1-final-offline-setup.sh лежит в /etc/uci-defaults/ и выполняется один раз при первой загрузке роутера. Весь ход пишется в /root/setup_log.txt. Многие шаги выполняются с учётом выбранного варианта.

  1. Инициализация. Логирование, определение версии ядра, архитектуры, модели роутера и текущего варианта сборки.
  2. База. Часовой пояс (Europe/Moscow), очистка строки FullCone, сохранение system.
  3. Пакетный менеджер. Определение apk/opkg, чистка репозиториев, добавление репозитория kmods (по версии ядра) и customfeeds.
  4. speedtest. Распаковка и установка бинаря в /usr/bin.
  5. hp_hiddifyПока не добавлен. Установка пакетов hiddify. Офлайн-установка движка + luci-app + i18n из /root/apps/; при наличии стоковой строки — dns_hijacked=1 (DNS остаётся за AGH); служба homeproxy выключается до ручной настройки.
  6. homeproxy (стоковые варианты на sing-box): форсится dns_hijacked=1, создаётся firewall-хелпер, патчится init, добавляется bypass-чейн nftables.
  7. Прокси под вариант. Настройка passwall2 / xray / v2rayA / podkop — в зависимости от выбранного варианта.
  8. youtubeUnblock. Запись правил nftables с ipset-байпасами, очередь 537, метка гостевой сети; START=99.
  9. internet-detector / owut. internet-detector выключается (чтобы не висел службой, а активировался при открытии страницы luci), owut настраивается.
  10. AdGuardHome. dnsmasq переезжает на порт 54, AGH занимает 53, создаётся LuCI-редирект на :8080, служба запускается.
  11. SQM и BBR. Если SQM установлен — настраивается CAKE; включается TCP BBR.
  12. FullCone NAT — только на ImmortalWrt и только при наличии модуля (на OpenWrt тихо пропускается).
  13. Время и идентичность. Синхронизация RTC/NTP, обновление баннера, установка hostname и commonname.
  14. switchРежим L2-коммутатора. Только для switch: все порты собираются в br-lan, DHCP-клиент, firewall-зоны и WAN отключаются.
  15. Перезагрузка. Отложенный ребут через 120 секунд в фоне — чтобы применить sysctl, AGH и остальное.
