Universal WRT Builder

Полный справочник по GitHub Actions workflow для компиляции OpenWrt/ImmortalWrt прошивок

Триггер: workflow_dispatch (вручную)
Runner: ubuntu-22.04
Timezone: Europe/Moscow
Jobs: setup + build (matrix)

Входные параметры запуска
все 8 параметров · workflow_dispatch
Параметр Тип По умолчанию Допустимые значения Описание
target choice combo
combo rb5009 nanopi-r6s nanopi-r5s slateax ax59u brume2 all
Целевое устройство. combo — жёстко заданная матрица из 3 jobs. all — запускает для rb5009 + nanopi-r6s с выбранным variant.
source choice openwrt
openwrt immortalwrt
Исходники: официальный OpenWrt или ImmortalWrt (китайский форк с расширенными пакетами).
branch choice openwrt-25.12
master openwrt-24.10 openwrt-25.12
master = snapshots (нестабильная). Стабильные ветки автоматически находят последний тег через GitHub API и фиксируют коммит.
variant choice standard
standard minimal clear switch crystal_clear podkop passwall v2raya xray
Набор пакетов. Определяет, что добавляется в .config и какие feeds подключаются. Подробно — в разделе 4.
download_binaries_agh boolean false true / false Скачать последний бинарник AdGuardHome (arm64) с GitHub и поместить в files/usr/bin/. Не работает для switch / clear / crystal_clear.
download_binaries_sb choice no
no latest 1.12.x
Скачать бинарник Sing-Box (arm64) напрямую с GitHub вместо версии из feed. 1.12.x — последний стабильный релиз из ветки LTS. Работает только для: standard, minimal, podkop, crystal_clear.
exclude_keywords string (пусто) Произвольная строка Пробел-разделённые ключевые слова. Все пакеты в .config, содержащие эти слова и отмеченные =y, будут принудительно отключены (is not set) перед make defconfig.
commit_hash string (пусто) Полный SHA коммита Опционально. Если указан — пропускает автопоиск тега через API и фиксирует именно этот коммит. Игнорируется для master.

Матрица сборки — что запускается
job: setup → build (matrix)
Как это работает Job setup генерирует JSON-матрицу, которую затем параллельно выполняет job build. Каждый элемент матрицы — отдельный runner с парой target + variant.

🔀 combo жёсткая матрица

nanopi-r6s minimal
rb5009 clear
rb5009 switch

Параметр variant полностью игнорируется

📦 all два устройства

nanopi-r6s выбранный variant
rb5009 выбранный variant

2 параллельных job

🎯 конкретное устройство

rb5009 / nanopi-r6s / nanopi-r5s / slateax / ax59u / brume2
+ variant = выбранный 1 job

1 job с выбранным variant

⚠ Важно: nanopi-r5s, slateax, ax59u, brume2 Эти устройства присутствуют в списке выбора target, но логика job setup обрабатывает только nanopi-r6s и rb5009 как часть списка all. При ручном выборе этих устройств создаётся 1 job, но конфигурация читается из файла ./config/conf/{target}, который должен существовать в репозитории.

Этапы пайплайна (steps)
порядок выполнения · job: build
1

Free up disk space

Удаляет: dotnet, Android SDK, Haskell, Azure CLI, Java JVM, Boost, Swift, Chromium, PowerShell.

Очищает Docker images (~3–4 ГБ). Удаляет браузеры, БД, mono через apt-get purge.

Всегда. Освобождает ~15–20 ГБ для сборки.

2

Checkout

Клонирует репозиторий самого workflow (actions/checkout@main).

Всегда.

3

Set permissions

Устанавливает chmod 755 на все файлы в files/usr/bin/ и files/etc/uci-defaults/ (если папки существуют).

Всегда.

4

Download AdGuardHome

Скачивает AdGuardHome_linux_arm64.tar.gz → files/usr/bin/AdGuardHome.

⚡ Условие: download_binaries_agh=true AND variant ∉ {switch, clear, crystal_clear}

5

Download Sing-box

Скачивает sing-box-*-linux-arm64.tar.gz → files/usr/bin/sing-box.

Версия latest — через /releases/latest. Версия 1.12.x — поиск через API с фильтром по тегу.

⚡ Условие: download_binaries_sb != 'no' AND variant ∉ {switch, clear, crystal_clear, passwall, v2raya, xray}

6

Configure Build Variables

Определяет: REPO_URL, REPO_BRANCH, TARGET_RELEASE, AUTO_COMMIT_HASH, KERNEL_PATH, DEVICE_CONFIG_COMMANDS.

Логика ветки: для master → branch=main (openwrt) / snapshots. Для стабильных — поиск последнего тега через GitHub API (или ручной коммит).

ImmortalWrt-специфика: проверяет HTTP 200 на папке релиза; если 404 → SNAPSHOT.

Фолбэк: 25.12 → releases/25.12.1 (openwrt) / releases/25.12-SNAPSHOT (immortalwrt); 24.10 → аналогично.

Всегда.

7

Init Env

Устанавливает зависимости сборки через apt-get. Список из внешнего скрипта + ручной полный список (ack, binutils, cmake, device-tree-compiler, jq, python3, и др.).

Устанавливает timezone = Europe/Moscow. Создаёт /workdir.

Всегда.

8

Clone Source

Клонирует в /workdir/openwrt: git clone --branch {REPO_BRANCH} {REPO_URL}.

Если задан AUTO_COMMIT_HASH — выполняет git checkout {hash} (точная версия).

Создаёт симлинк {GITHUB_WORKSPACE}/openwrt → /workdir/openwrt.

Всегда.

9

Apply custom patches

Ищет *.patch файлы в директории patches/. Применяет каждый через patch -p1 -d openwrt.

Если папки нет или пуста — пропускается без ошибки.

⚡ Условно: только если файлы *.patch существуют в patches/

10

Update Feeds & Config

Комплексный шаг. Порядок:

① Копирует feeds.conf.default (если есть) → ② Запускает WRT-part1.sh (добавление репозиториев) → ③ Добавляет variant-специфичные feeds → ④ Заменяет зеркала git.openwrt.org на github.com → ⑤ feeds update -a → ⑥ WRT-part2.sh (фиксы версий, строго между update и install) → ⑦ feeds install -a → ⑧ Копирует WRT.config и files/ → ⑨ Применяет DEVICE_CONFIG_COMMANDS → ⑩ WRT-part3.sh → ⑪ Модификации .config по variant → ⑫ WRT-part4.sh → ⑬ Логика exclude_keywords → ⑭ make defconfig.

Всегда.

11

Kernel Hash

Получает актуальный хеш ядра с официального сервера загрузок и применяет в include/kernel-defaults.mk.

Это нужно для совместимости с официальными kmod-пакетами из репозитория.

Подробно — раздел 8.

Всегда.

12

Compile

① Попытка: make -j$(nproc + 1) (параллельно). ② При ошибке: make -j1 V=s > build.log 2>&1 (verbose, однопоточно). При повторной ошибке: показывает последние 200 строк лога, сохраняет bin/build_failure.log, завершается с кодом 1.

Всегда.

13

Save Bootloader files (Deep Search)

Ищет и копирует в bin/u-boot-files/:

— Rockchip (R5S/R6S): idbloader.img, u-boot.itb из build_dir/target-*

— MediaTek (Filogic/SlateAX/AX59U): *.fip, u-boot.bin из папок u-boot

— Дополнительно: *idbloader.img, *u-boot.itb, u-boot.elf из bin/targets

Всегда (ищет, не падает если не найдено).

14

Upload artifact

Загружает всю папку openwrt/bin (прошивки + kmod пакеты + u-boot файлы + лог при ошибке) как GitHub Actions artifact.

Имя: {source}_{branch}_{variant}_{target}_{YYYYMMDDHHMM}

Retention: 10 дней. Действие: actions/upload-artifact@v7.

Всегда.

15

Delete workflow runs

Удаляет старые запуски workflow старше 10 дней через gh run delete.

Лимит выборки: 100 последних запусков. Использует GITHUB_TOKEN.

Всегда выполняется

8
Логика Kernel Hash
совместимость с официальными kmod-репозиториями
Зачем это нужно OpenWrt включает хеш ядра в имена kmod-пакетов. Если хеш отличается от официального (собранного на серверах OpenWrt), то пакеты из официального репозитория нельзя будет установить через opkg. Скрипт синхронизирует хеш с официальными серверами ещё до сборки.
ШагЧто происходит
1. Домен immortalwrt → immortalwrt.kyarucloud.moe (зеркало, т.к. основной нестабилен)
openwrt → downloads.openwrt.org
2. Primary URL https://{domain}/{TARGET_RELEASE}/targets/{KERNEL_PATH}/kmods/
Парсит список, ищет паттерн 6.x.x-N-{hash}
3. Fallback Если Primary пуст И ветка не master/main → пробует releases/{version}-SNAPSHOT/targets/{KERNEL_PATH}/kmods/
4. Generic fallback Если паттерн 6.x не найден → пробует любой паттерн N.N.N-N-{hash}
5. Выбор хеша Из всех найденных версий берёт наибольшую (сортировка по major.minor.patch). Извлекает только хеш (3-я часть после дефиса).
6. Применение Найден: sed -i "s/$(MKHASH) md5/echo "{hash}"/g" include/kernel-defaults.mk
Не найден: файл не меняется, OpenWrt генерирует хеш сам (kmod из репо работать не будут)

многопоточная → однопоточная fallback
ПопыткаКомандаПоведение при ошибке
