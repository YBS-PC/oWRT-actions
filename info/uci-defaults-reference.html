8. Настройка youtubeUnblock

УсловиеФайлЧто делается
ВСЕГДА
(для каждого из трёх ipset)		 /etc/config/firewall Проверяется наличие ipset-секций: dpi_ips, no_dpi_ips, dpi_guest_ips. Если какой-то нет — создаётся пустой ipset с match=net, family=ipv4.
ВСЕГДА /usr/share/firewall4/templates/ruleset.uc Строка meta l4proto { tcp, udp } flow offload @ft; заменяется на meta l4proto { tcp, udp } ct original packets ge 30 flow offload @ft;. Flow offload включается только после 30 пакетов, давая youtubeUnblock время обработать начало соединения.
УСЛОВИЕ: /usr/bin/youtubeUnblock существует и исполняем /usr/share/nftables.d/ruleset-post/537-youtubeUnblock.nft Файл создаётся или перезаписывается целиком. Содержимое:
1. Блокировка QUIC (UDP 443 к dpi_ips → reject)
2. Цепочка youtubeUnblock в postrouting, mangle-1
3. mark 0x42 (гостевая сеть) → return
4. no_dpi_ips → return (исключения, сюда добавлять IP VPN-серверов)
5. dpi_ips + TCP 443 + <20 пакетов → NFQUEUE 537
6. dpi_ips + UDP + <9 пакетов → NFQUEUE 537
7. output: mark 0x8000 → accept (уже обработанные youtubeUnblock)
УСЛОВИЕ: youtubeUnblock установлен автозапуск youtubeUnblock включается в автозапуск (enable)
⚠ guest_mark.nft закомментирован в скрипте — НЕ создаётся. Гостевая сеть через метку 0x42 работать не будет до раскомментирования.

14. Режим коммутатора (switch)

УсловиеФайлЧто делается
ВАРИАНТ: switch
+ lock-файл ещё не создан 		/etc/config/network Удаляются интерфейсы lan, wan, wan6.
Все физические порты (кроме системных, Wi-Fi, br-, tun, sit и на DSA-устройствах — eth0) объединяются в мост br-lan.
Настройки моста: stp=1, igmp_snooping=1, multicast_querier=1, ipv6=0.
LAN получает IP по DHCP (proto=dhcp).
packet_steering=2, удаляется ula_prefix.
ВАРИАНТ: switch
+ lock-файл ещё не создан 		/etc/config/dhcp lan.ignore=1 — DHCP-сервер на LAN отключён.
lan.dhcpv6=disabled, lan.ra=disabled.
odhcpd.disabled=1, odhcpd отключается из автозапуска.
Удаляется секция dhcp.wan.
ВАРИАНТ: switch
+ lock-файл ещё не создан 		/etc/config/firewall Удаляется зона WAN (zone[1]).
Удаляется forwarding[0].
Удаляются все rule-секции (while uci delete firewall.@rule[0]).
Зона LAN: input/output/forward = ACCEPT.
Включается аппаратное ускорение: flow_offloading=1, flow_offloading_hw=1.
ВАРИАНТ: switch hostname Hostname меняется на МОДЕЛЬ-switch вместо МОДЕЛЬ-oWRT (например RB5009-switch)
⚠ DSA-определение: если в /sys/class/net/ есть порты вида p1, p2… — это DSA-архитектура (RB5009), eth0 исключается из моста. Для R6S такие порты есть, поэтому eth0 исключается.
